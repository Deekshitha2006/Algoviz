# 🤝 Contributing to AlgoViz

Thank you for your interest in contributing! Here's everything you need to know.

---

## 🐛 Reporting a Bug

1. Go to [Issues](https://github.com/deekshitha2006/Algoviz/issues)
2. Click **New Issue**
3. Include:
   - **Description** — what went wrong
   - **Steps to reproduce** — exactly what you did
   - **Expected vs actual** — what should have happened vs what did
   - **Device & browser** — e.g. iPhone 14 / Safari, Android / Chrome, Windows / Firefox

---

## 💡 Suggesting a Feature

Open an Issue with the label `enhancement` and describe:
- What you'd like to see
- Why it would be useful
- Any ideas on how it could work

---

## 🔧 Submitting a Pull Request

1. **Fork** this repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR-USERNAME/Algoviz.git
   cd Algoviz
   ```
3. **Create a branch** with a descriptive name:
   ```bash
   git checkout -b fix/mobile-context-menu
   # or
   git checkout -b feature/astar-algorithm
   ```
4. **Make your changes** — keep them focused and minimal
5. **Test** your changes in both desktop and mobile (or browser DevTools mobile view)
6. **Commit** with a clear message:
   ```bash
   git commit -m "fix: long-press context menu closes immediately on iOS"
   ```
7. **Push** to your fork:
   ```bash
   git push origin your-branch-name
   ```
8. Open a **Pull Request** on the main repo and describe what you changed and why

---

## 📐 Code Style

- Vanilla JS only — no frameworks or libraries
- Keep variable names consistent with the existing code
- Comment any non-obvious logic
- Test on both grid mode and custom graph mode

---

## ✅ Good First Contributions

- Fix a reported bug
- Improve mobile UX
- Add a new algorithm (A*, Bidirectional BFS, etc.)
- Improve accessibility (keyboard navigation, ARIA labels)
- Add more code examples (Java, TypeScript, etc.)
- Improve the Compare page

---

## 📬 Questions?

Open an Issue or reach out via GitHub. All contributions, big or small, are appreciated!
